## TCP 와 UDP 방식을 비교 설명해주세요.

```markdown
TCP 프로토콜과 UDP 프로토콜은 TCP/IP 4계층의 전송계층에 해당되는 프로토콜입니다

TCP는 가상회선방식을 통한 연결지향적 프로토콜로써
3-way handshake를 통해 논리적 연결을 한 후, 데이터를 전송하며
데이터 전송이 완료되면 4-way handshake를 통해 연결을 해제합니다
따라서 TCP 프로토콜은 데이터의 신뢰성이 보장되며 
TCP 세그먼트 내부의 있는 데이터 순서 값으로 전송 데이터의 순서가 보장이됩니다
따라서 TCP 프로토콜은 중요한 데이터를 통신하며 데이터 누락이 없어야 하는 애플리케이션에서 사용되기 적합합니다

반대로 UDP 같은 경우에는 데이터그램 방식을 통한 비연결형 프로토콜로써
별다른 3-way handshake 같은 데이터 전송 전처리 작업을 하지 않고 Header에 값도 적기 때문에
데이터의 신뢰성이 보장되지는 않지만 네트워크 부하를 감소할 수 있으며 전송속도가 TCP 프로토콜에 비해서 비교적 빠릅니다
따라서 중요한 데이터를 통신하는 것이 아닌 데이터 누락을 감수하더라도 데이터 전송속도가 빨라야 하는
실시간 동영상 스트리밍 서비스 같은 곳에 사용하기 적합합니다
실시간 동영상 스트리밍 서비스는 일부 데이터 누락이 되더라도 사용자는 잘 인식하지 못하므로
전송 처리 속도가 빠른 UDP 프로토콜을 사용하는 것이 좋습니다
```


## 서비스에서 1가지 기능에 대한 통신 프로토콜을 TCP -> UDP 로 변경해야한다고 할 때, 어떤 기능의 프로토콜을 변경하는 게 좋을까요 ?
```markdown
만약 TCP 프로토콜로 사용되어 있던 것을 UDP로 변경해야 할 경우에는
애플리케이션 기능 중 데이터 손실을 감수하고서라도 데이터 전송 처리속도가 빨라야 하는 서비스에 적용할 것입니다
중요한 데이터를 통신하는 서비스에서는 TCP를 사용해야 하지만 동영상 스트리밍 서비스 같은 라이브로 끊김이 없는 서비스가 필요할 시
UDP 프로토콜이 적합하므로 스트리밍 서비스 기능을 UDP로 변경할 것 같습니다
```
