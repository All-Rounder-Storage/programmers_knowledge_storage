## HTTP 응답코드의 종류는 어떻게 되는지 설명해주세요
```markdown
HTTP의 응답코드는 크게 5가지로 구분됩니다.
1xx번대는 요청 정보 처리중인 상태이지만 잘 쓰지 않구요
2xx번대는 성공 번대입니다.
자주 사용하는 것중엔 200과 201이 있는데요
200은 말그대로 해당 요청에 대해서 성공했다는 의미이고,
201은 생성이 성공했다는 의미로 Response 헤더에 Location으로 생성된 리소스의 위치를 반환할 수 있는 특징이 있습니다.

3xx번대는 Redirection 코드로 요청 완료가 되기 전에 추가적인 작업이 필요함을 의미합니다.
자주 사용되는 코드는 302 코드인데요
초기 스펙에서 302에 의도는 요청 메서드가 유지되는것이였지만 대부분의 브라우저에서 메서드를 GET으로 변경되게끔 구현되었습니다.
따라서 302는 현재 메서드가 GET으로 변경되고 본문내용이 삭제될수 있음 이라는 다소 애매한 코드가 되었습니다.
대부분의 브라우저에서는 GET으로 변경되고 본문내용이 삭제되지만 일부 브라우저에서는 그렇지 않은 경우가 있는데요
따라서 최근 스펙에서는 303과 307을 사용할 것을 권장합니다.
303은 확실하게 GET으로 변경되고 본문내용은 삭제되는 코드이구요
307은 확실하게 요청 메서드와 본문이 유지되는 특징이 있습니다.
결론은 요청이 GET으로 변경되고 본문이 삭제되도 되는 것이라면 302를 써도 무방합니다.

400번대와 500번대에 대해서는 클라이언트 오류냐, 서버의 오류나의 차이로 나눠지는데요
해당 내용은 아래에서 기재하겠습니다.
```
    
## HTTP 4xx 에러 vs HTTP 5xx 에러 차이점에 대해 설명해주세요
```markdown
4xx번대와 5xx번대의 차이점은 오류 주체가 클라이언트인지, 서버인지로 나눠집니다.

4xx번대 에러는 클라이언트 오류입니다.
클라이언트에서 요청 자체를 잘못 보내 서버에서 요청을 처리할 수 없을때 사용되는 코드입니다.
자주 사용되는 코드는 400, 401, 403, 404 등이 있는데요

400번은 Bad Request로 요청 자체가 잘못 되어 서버가 요청을 처리하지 못할 때 사용되는 코드입니다.
이 경우 클라이언트는 데이터를 변경하여 다시 요청해야 합니다.

401은 인증이 되지 않을 때, 즉 로그인이 되지 않아 서버측에서 본인이 누구인지 판단하지 못할 때 사용되는 코드입니다.

403은 인증이 돼서 사용자가 누구인지 알지만, 권한이 없어 접근이 제한될 때 사용됩니다
Admin 페이지 같은것을 예로 들 수 있습니다.

404의 경우는 요청한 리소스가 존재하지 않을때 사용되는 코드입니다.
말 그대로 해당 메서드의 해당 path는 서버에서 지원하지 않는 리소스입니다.
또한 403 같이 권한이 제한되어 리소스 자체를 숨길때도 404를 사용하곤 합니다.

5xx번대 에러는 서버 오류입니다.
자주 사용되는 코드는 단연 500 Internal Server Error이구요, 추가로 503 코드도 있습니다.
500번 에러는 클라이언트 요청은 정상적으로 올라왔지만 서버단에서 요청을 제대로 처리 완료하지 못할 때 사용되는 코드입니다.
따라서 서버 오류 중 애매하면 모두 500번 에러로 반환하면 됩니다.
간혹 비지니스적인 제한으로 500을 떨어트리는 경우도 있는데요
email 중복같은 부분이나 나이제한으로 구매를 정상적으로 못할때 등 프로세스 적인 오류는 500번이 아닌 4xx 대로 반환하여야 합니다.
email 중복은 409 Conflict가 적절할 것 같고 나이제한은 400 Bad Request나 403 권한제한 등을 사용하면 될 것 같다는 생각입니다.

추가로 503 같은 경우는 일시적으로 과부하가 걸리거나 예정된 작업이 진행 될 때 사용되는 코드입니다.
따라서 Response 헤더에 Retry-After 필드로 복구 예정시간을 반환할 수 있다는 특징이 있습니다.


```
