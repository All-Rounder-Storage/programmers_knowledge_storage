## 1. HTTP request method 중 GET vs POST에 대한 비교 설명해주세요

```markdown
GET 과 POST는 대표적인 조회와 등록을 위한 메서드입니다.

GET은 쿼리스트링을 통해 전달할 데이터를 담아서 전송하는 특징이 있습니다.
반면 POST는 request body에 전달할 데이터를 json 형식으로 담아서 전송하는 특징이 있습니다.

추가로 GET은 최근 HTTP 버전에서 request body에 데이터를 담아서 보낼 수 있게 변경되었지만
라이브러리 및 프로그램에서 해당 기능을 제공하지 않는 곳이 많기 때문에 웬만하면 사용하지 않는 것이 바람직하다고 생각합니다.
```
 


## 2. HTTP request method 중 PUT vs PATCH에 대한 비교 설명해주세요

```markdown
PUT과 PATCH의 차이는 리소스 전체수정인지, 부분수정인지로 나눠집니다.

PUT 같은 경우에는 리소스를 덮어씌우기에 개념으로 보면 됩니다.
수정할 데이터가 존재하지 않을 경우에는 새로 등록을 하고, 존재할 경우에는 request body에 담긴 데이터로 덮어 씌우기가 됩니다.
따라서 만약 name과 age라는 리소스가 존재하지만 request body 데이터에 age만 존재한다면 name이라는 리소스는 사라지게 됩니다.

PATCH는 부분수정이 가능하므로 request body 데이터에 age만 있어도 name은 그대로 유지되며 age 값만 덮어씌워지게 됩니다.
```


## 3. HTTP 메서드에 대해서 설명해주세요

```markdown
HTTP 메서드는 크게 9가지가 있고 그중에 주로 사용되는 메서드는 5가지로 GET, POST, PUT, PATCH, DELETE가 있습니다.

GET은 리소스를 조회할때 사용됩니다.
POST는 주로 리소스를 등록할때 사용되고, 여러가지 동작이 맞물려 메서드를 선택하기 애매할 땐 POST를 주로 사용합니다.
PUT도 마찬가지로 등록의 기능도 있지만 주요 동작은 리소스 전체수정입니다. 
PATCH는 리소스를 부분 수정할 때 사용하는 메서드이구요
DELETE는 리소스를 삭제할때 사용되는 메서드입니다.

GET과 DELETE는 주로 쿼리 파라미터로 전송할 데이터를 담아 보내고
나머지 POST, PUT, PATCH의 경우는 request body에 데이터를 담아 보냅니다.

메서드의 특징은 크게 3가지로 볼 수 있는데요
안전성, 멱등, 캐시가능여부 등으로 나뉠 수가 있습니다

안전성은 리소스를 요청했을때 그 리소스의 값이 변함이 없는지에 대한 기준인데요
GET과 HEAD만 안전성이 보장되는 메서드입니다.

멱등은 같은 요청을 여러번 하였을 경우 그 결과값이 변함이 없는지에 대한 기준인데요
POST와 PATCH 는 멱등하지 않고 나머지 메서드는 멱등합니다.
POST 같은 경우에는 등록을 중복으로 하면 계속 쌓여서 호출을 하면 할수록 데이터가 계속 늘어나는 특징이 있고
PATCH 같은 경우에는 다양한 기능을 제공하여 특정 리소스를 증가시키는 데이터를 담아 보낼 수도 있습니다.
따라서 요청을 하면 할수록 값이 변경되고 쌓이기 때문에 두가지 메서드의 경우 멱등하지 않습니다
이 멱등의 특징을 활용하여 대표적으로 자동 복구 기능을 예로 들 수 있습니다.

캐시가능여부는 응답결과 리소스를 캐시하여 사용해도 되는지에 대한 기준인데요
GET, HEAD, POST, PUT 등이 가능하지만 실제로는 GET과 HEAD만 사용하는 것을 추천합니다.
POST와 PUT은 캐시를 하려면 request body 값까지 고려하여 설계하여야 하지만 사실상 쉽지 않아 잘 사용되지 않습니다.


```
