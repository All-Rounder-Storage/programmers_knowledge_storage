## URL 에 www.example.com 을 쳤을 때 일어나는 일들을 설명해주세요
```markdown
1. 해당 URL을 들고 DNS 서버로 접속합니다
2. DNS 서버에서 해당 도메인을 호스팅하고 있는 IP 주소를 받아옵니다
3. IP 주소를 통해 TCP Connection을 요청합니다 
4. 7계층 응용 계층에서 사용자가 입력한 주소 정보를 들고 하위 계층을 들어갑니다  
5. 6계층 표현 계층에서는 사용자가 입력한 데이터 형식을 확인하고 정보를 인코딩을 하며, 데이터의 암호화를 진행하고 하위 계층으로 전달합니다
6. 5계층 세션 계층에서는 목적지 서버에 세션 연결을 설정하고 하위계층으로 전달합니다 
7. 4계층 전송 계층에서는 TCP 연결일 경우 3-way handshake를 진행하여 목적지 서버가 정상인지 확인하고,
   출발지와 목적지의 port 정보 및 신뢰성 있는 데이터를 위한 정보들(전송 제어, 순서, 검증정보 등)을 캡슐화 하여
   TCP 세그먼트를 생성하고 하위 계층으로 전달합니다
   UDP 프로토콜인 경우 바로 출발지와 목적지의 port 정보와 체크섬 정도만으로 구성된 세그먼트를 생성뒤 하위계층으로 바로 전달합니다\
8. 3계층 네트워크 계층에서는 출발지와 목적지의 IP, 전송 데이터를 캡슐화 하여 IP 패킷을 생성 후 세그먼트에 붙인 후에
   IP 주소를 통해 해당 목적지가 있는 곳까지 라우터가 포워딩을 통해 라우팅이 진행됩니다
   포워딩 과정에서 3계층 ~ 1계층이 반복됩니다
9. 2계층 데이터 링크 계층에서는 스위치의 ARP Table을 통해 목적지의 Mac을 탐색합니다
   스위치에 연결되어 있는 Mac이 발견될때까지 계속 이동합니다
   또한 1계층에 전달하기전 Framing을 통해 데이터 앞 뒤에 구분자를 넣어주며 프레임 데이터를 하위 계층으로 전달합니다
10. 1계층 물리계층에서는 두 대의 기기끼리 인코딩과 디코딩을 통해 데이터를 전달합니다
   인코딩을 통해 데이터를 전기 신호(아날로그 신호)로 변경하여 전송하고 수신측은 전기신호(아날로그 신호)에서 데이터로 변환하는 디코드 과정을 반복합니다.
    
목적지에 잘 도착하면 해당 PC에서 위의 과정을 역순으로 진행되어 최상위 계층인 응용 계층을 통해 HTML이 보여집니다
```

## 3-Way handshake & 4-Way handshake에 대해 간략하게 설명해주세요
```markdown
전송계층에서 TCP 프로토콜을 사용하는 경우 데이터 전송과는 별개로 두가지의 작업이 수행됩니다
3-way handshake는 데이터 전송 전 신뢰성 있는 데이터 전달을 위한 작업으로 목적지 서버의 세션 상태를 확인하는 과정입니다 (잘 살아 있는지)
1. 발신 측에서는 SYN을 통해 세션 연결 요청을 진행합니다
2. 수신 측에서는 요청 수락인 ACK 과 더불어 SYN을 보냅니다
3. 발신 측에서는 ACK을 보냅니다 (마지막 ACK 전송 시 요청 데이터를 같이 전달 할 수 있습니다)
3-1. 이 말은 즉슨 마지막 ACK 데이터와 함께 본 데이터를 같이 들고 하위계층인 3계층으로 들어가서 라우팅 작업이 시작된다는 것입니다
     매 데이터 전송마다 라우팅 경로는 같지 않습니다

4-way handshake는 데이터 전송이 모두 끝난 후 세션 연결을 끊을 목적으로 수행되는 작업입니다
1. 발신측에서는 세션 연결 종료 요청인 FIN을 보냅니다
2. 수신측에서는 세션 연결 종료 수락인 ACK을 보냅니다
3. 수신측에서는 발신측에서 전송 된 데이터가 누락되지 않게 조금의 시간을 기다린 뒤 FIN을 보냅니다
4. 발신측은 ACK을 통해 양쪽간의 세션 연결은 종료됩니다
```
