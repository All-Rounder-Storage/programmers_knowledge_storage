## OAuth2.0의 흐름에 대해 간단히 설명해주세요
```markdown
OAuth2.0은 4가지의 권한 부여 방식이 존재하지만 일반적인 흐름으로 설명하겠습니다.

사용자는 인증을 한 후, 인가를 완료하여야 자원 리소스에 접근 할 수 있습니다.

본 설명에서는 OAuth 2.0에서 가장 많이 사용되는 Authorization code grant 방식을 설명하겠습니다.
1. 사용자가 클라이언트를 통해 서버로 접근합니다.
2. 서버는 인증(Authentication) 페이지로 이동시킵니다.
3. 사용자가 인증에 성공하면 인가 코드와 redirect url을 서버에서 반환합니다.
4. 클라이언트는 redirect url을 통해 인가코드를 사용하여 Access Token을 요청합니다.
5. 인가(Authorization) 서버에서는 인가코드를 확인 후 Access Token과 Refresh Token 등을 반환합니다.
6. 클라이언트는 로그인 성공 후 페이지를 보여주며 이후엔 보호된 자원 리소스에 접근이 가능하여 사용자는 서비스를 이용할 수 있습니다.
```
